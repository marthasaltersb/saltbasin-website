# Deploying Salt Basin Net Works to Render

This guide deploys the full app (Node backend + built React frontend, one service) to Render's free tier in about 15 minutes. SQLite data and uploads are ephemeral on free; instructions for adding persistent storage are at the bottom.

## Prerequisites

- A [Render account](https://render.com) (free, GitHub sign-in works)
- The repo pushed to GitHub
- Optional: an [Anthropic API key](https://console.anthropic.com) for the editor agent and BestyStaff (not required for v1)

## Step 1 — Push the repo

If you haven't already:

```powershell
cd C:\Users\mbets\saltbasin-website
git init
git add .
git commit -m "Initial Salt Basin website"
gh repo create saltbasin-website --private --source=. --push
```

Or push to an existing remote if you already have one.

## Step 2 — Create the Render service

1. In the Render dashboard, click **New +** → **Blueprint**.
2. Connect your GitHub account if you haven't already.
3. Pick the `saltbasin-website` repo.
4. Render reads `render.yaml` and shows the proposed service. Click **Apply**.

Render will start the first build immediately. It runs `npm install && npm run build` (which produces `dist/`) and then `npm start`. Expect 3–5 minutes on the first deploy.

## Step 3 — Set the required secrets

When prompted (or via the service's **Environment** tab afterward), set:

| Variable | Value |
|---|---|
| `ADMIN_EMAIL` | Your admin email (e.g. `betsy@saltbasin.net`) |
| `ADMIN_INITIAL_PASSWORD` | The initial password — pick something strong, change it on first login |
| `ANTHROPIC_API_KEY` | Leave blank for now. Set when the editor agent goes live. |

`SESSION_SECRET` is auto-generated by Render.

## Step 4 — Visit the site

Render gives you a `.onrender.com` URL like `https://saltbasin-website.onrender.com`. Open it:

- Public site: the root URL
- Admin login: `<your-url>/admin/login`

Sign in with the email and password you set in Step 3. Change the password from the admin once you're in (Phase 3 admin tooling).

## Step 5 — (Optional) Custom domain

In the Render service's **Settings** → **Custom Domains**, add `saltbasin.net` (or whichever domain you own). Render gives you DNS records to set at your registrar. HTTPS is automatic via Let's Encrypt.

After the domain is live, set `FRONTEND_ORIGIN=https://saltbasin.net` in the env vars and redeploy — that locks CORS down to your domain only.

## Step 6 — (Optional) Persistent storage

The free tier wipes SQLite data and uploaded photos on every redeploy. To make them persist:

1. Upgrade the service to the **Starter** plan ($7/mo at time of writing).
2. Uncomment the `disk:` block at the bottom of `render.yaml`.
3. Commit and push. Render attaches the disk on the next deploy.

The disk is mounted at `/opt/render/project/src/server/data`, which is where SQLite and uploads already live.

## Troubleshooting

**"`dist/` is missing" warning at boot** — the build didn't run. Check the Render build logs; usually a Node version mismatch or a missing dependency.

**Login returns 401 but credentials are right** — `secure: true` cookies require HTTPS. Render serves HTTPS by default, so this should work; if you're hitting `http://...onrender.com` directly, switch to `https://`.

**Uploads upload but the image doesn't show on the public site** — confirm you clicked **Publish** in the admin after saving the draft. Public only reads the published snapshot.

**Sessions don't persist across redeploys (free tier)** — expected. The sessions table lives in the ephemeral SQLite. Re-login.

**Service is slow on first hit after idle** — expected on free tier. Render spins services down after 15 min of inactivity; first request triggers a cold start (~30 s). Use a paid plan or a small uptime pinger if it matters.

## Going to production

Before pointing real traffic at this:

1. Set a strong `ADMIN_INITIAL_PASSWORD` and change it on first login.
2. Add a custom domain and set `FRONTEND_ORIGIN`.
3. Add the persistent disk (you don't want to lose user data).
4. Enable the pre-launch landing gate in the admin while iterating.
5. Set up Render's automatic database backups (only available with the disk add-on).
6. Wire `ANTHROPIC_API_KEY` once the editor agent / BestyStaff are enabled.
